# Desloppify Issues Skill

This skill converts desloppify scan output into structured Multica issues. It consumes the normalized JSON artifact produced by `devtools desloppify` and creates one issue per group of findings.

## Prerequisites

Run the scan first and save the output:

```bash
devtools desloppify --repo <path> --output desloppify-results.json
```

## Grouping Strategy

Choose how to group findings into issues before creating them. The three modes are:

| Mode | Best for |
|---|---|
| `detector` | One issue per violation type across the repo — good for systematic fixes |
| `file` | One issue per file — good for per-file code reviews |
| `package` | One issue per monorepo package — good for package-level ownership |

Default: **detector**.

## Creating Issues

Read the JSON output and create issues using `multica issue create`. The standard pattern groups by detector and targets T1 (critical) and T2 (high) findings:

```bash
# Parse findings and create issues
python3 - <<'EOF'
import json, subprocess, sys, tempfile, os

INPUT = "desloppify-results.json"
PARENT_ISSUE_ID = ""   # optional: link to initiative (e.g. APR-65)
PROJECT_ID = ""        # optional: assign to a project
MIN_TIER = 2           # include T1 and T2
DRY_RUN = False        # set True to preview without creating

with open(INPUT) as f:
    result = json.load(f)

repo = result["repo"]

# Collect open findings up to MIN_TIER
findings = []
for pkg in result["packages"]:
    for issue in pkg["issues"]:
        if issue["tier"] <= MIN_TIER and issue["status"] == "open":
            findings.append({**issue, "_pkg": pkg["name"]})

if not findings:
    print(f"No T1-T{MIN_TIER} open findings. Nothing to create.")
    sys.exit(0)

# Group by detector
groups = {}
for f in findings:
    key = f["detector"]
    groups.setdefault(key, []).append(f)

TIER_LABELS = {1: "T1-critical", 2: "T2-high", 3: "T3-medium", 4: "T4-low"}
PRIORITIES  = {1: "critical", 2: "high", 3: "medium", 4: "low"}

for detector, items in groups.items():
    min_tier = min(i["tier"] for i in items)
    tier_label = TIER_LABELS.get(min_tier, f"T{min_tier}")
    priority = PRIORITIES.get(min_tier, "medium")
    count = len(items)

    title = f"[desloppify][{repo}] {detector}: {count} issue{'s' if count > 1 else ''} ({tier_label})"

    file_lines = "\n".join(f"- `{i['file']}`: {i['message']}" for i in items)
    description = f"""## desloppify findings — {detector}

**Repo:** {repo}
**Detector:** {detector}
**Count:** {count}
**Severity:** {tier_label}

### Affected files

{file_lines}

---
*Auto-generated by desloppify-issues skill*"""

    if DRY_RUN:
        print(f"[DRY RUN] Would create: {title} ({priority})")
        continue

    with tempfile.NamedTemporaryFile(mode="w", suffix=".md", delete=False) as tmp:
        tmp.write(description)
        tmp_path = tmp.name

    try:
        cmd = ["multica", "issue", "create",
               "--title", title,
               "--priority", priority,
               "--description-file", tmp_path]
        if PARENT_ISSUE_ID:
            cmd += ["--parent", PARENT_ISSUE_ID]
        if PROJECT_ID:
            cmd += ["--project", PROJECT_ID]

        out = subprocess.run(cmd, capture_output=True, text=True)
        if out.returncode == 0:
            print(f"Created: {title}")
            if out.stdout.strip():
                print(f"  → {out.stdout.strip()}")
        else:
            print(f"FAILED: {title}\n  {out.stderr.strip()}", file=sys.stderr)
    finally:
        os.unlink(tmp_path)
EOF
```

## Full Workflow

1. **Run the scan** and save the artifact:
   ```bash
   devtools desloppify --repo . --output desloppify-results.json
   ```

2. **Preview** what would be created (`DRY_RUN = True` in the script above).

3. **Set parameters** in the script:
   - `INPUT` — path to the JSON artifact
   - `PARENT_ISSUE_ID` — parent issue to link findings under (e.g. the Code Quality Sentinel initiative)
   - `PROJECT_ID` — project to assign issues to
   - `MIN_TIER` — `1` for critical only, `2` for critical+high (recommended), `3` for medium+

4. **Run** to create issues.

## Linking to an Initiative

To link all findings to the Code Quality Sentinel initiative (APR-65), set:

```python
PARENT_ISSUE_ID = "<uuid of APR-65>"
```

Get the UUID with:
```bash
multica issue get APR-65 --output json | python3 -c "import json,sys; print(json.load(sys.stdin)['id'])"
```

## Tips

- Run with `DRY_RUN = True` first to review groupings and titles before committing.
- Findings with `"status": "resolved"` or `"status": "ignored"` are automatically skipped.
- For monorepos, consider grouping by `package` so each package owner gets a focused issue.
- Re-running the script on the same scan file is idempotent only if you deduplicate by title first — otherwise you may create duplicate issues.
