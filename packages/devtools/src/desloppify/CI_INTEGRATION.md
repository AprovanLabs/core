# CI Integration: desloppify quality gate

This guide shows how to wire `desloppify scan --profile ci` into GitHub Actions
as a PR quality gate using the `@aprovan/devtools` runner.

## Prerequisites

- Python 3.11+ with `desloppify[full]` installed
- Node.js 20+ with `@aprovan/devtools` installed
- `multica` CLI (optional, for issue creation on failure)

## Basic quality gate

Add this step to your PR workflow to fail when the strict score drops below a
threshold:

```yaml
name: Code Quality Gate

on:
  pull_request:

jobs:
  desloppify:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.13"
      - run: pip install "desloppify[full]"
      - uses: actions/setup-node@v4
        with:
          node-version: "20"
          cache: pnpm
      - run: pnpm install --frozen-lockfile
      - name: Run desloppify scan
        run: |
          desloppify scan --profile ci --skip-slow
          SCORE=$(desloppify status --json | python3 -c "import json,sys; print(json.load(sys.stdin)['strict_score'])")
          echo "Strict score: $SCORE"
          THRESHOLD=60
          if [ "$(echo "$SCORE < $THRESHOLD" | bc -l)" -eq 1 ]; then
            echo "::error::Strict score $SCORE is below threshold $THRESHOLD"
            exit 1
          fi
```

## Monorepo per-package scan

For monorepos with multiple packages, use the `devtools desloppify-run` command
to scan each package with isolated state:

```yaml
name: Code Quality Gate

on:
  pull_request:

jobs:
  desloppify:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.13"
      - run: pip install "desloppify[full]"
      - uses: actions/setup-node@v4
        with:
          node-version: "20"
          cache: pnpm
      - run: pnpm install --frozen-lockfile
      - name: Run per-package scan
        run: |
          npx devtools desloppify-run \
            --repo . \
            --profile ci \
            --output desloppify-results.json
      - name: Check thresholds
        run: |
          python3 -c "
          import json, sys
          result = json.load(open('desloppify-results.json'))
          failed = False
          for pkg in result['packages']:
              if pkg['score']['strict'] < 60:
                  print(f\"::error file={pkg['path']}::Package {pkg['name']} strict score {pkg['score']['strict']} is below threshold 60\")
                  failed = True
              else:
                  print(f\"Package {pkg['name']}: strict score {pkg['score']['strict']} OK\")
          if failed:
              sys.exit(1)
          "
      - uses: actions/upload-artifact@v4
        if: always()
        with:
          name: desloppify-results
          path: desloppify-results.json
```

## Creating Multica issues on failure

To automatically create tracking issues for T1/T2 findings:

```yaml
      - name: Create Multica issues for critical findings
        if: failure()
        run: |
          npx devtools desloppify-issues \
            --input desloppify-results.json \
            --repo my-repo \
            --parent <initiative-issue-id> \
            --project <project-id> \
            --min-tier 2 \
            --group-by detector \
            --dry-run
        env:
          MULTICA_API_KEY: ${{ secrets.MULTICA_API_KEY }}
```

Remove `--dry-run` to actually create issues.

## Configuration options

| Option | Description | Default |
|---|---|---|
| `--profile ci` | Fast scan profile that skips slow detectors | objective |
| `--skip-slow` | Skip slow detectors (dupes) even in full profile | off |
| `--repo <path>` | Repository root path | required |
| `--packages <paths...>` | Explicit package paths (auto-detected if omitted) | auto |
| `--exclusions <patterns...>` | Additional exclusion patterns beyond defaults | none |
| `--state-dir <path>` | Custom state directory | `<repo>/.desloppify` |
| `--min-tier <n>` | Maximum tier to include for issue creation (1-4) | 2 |
| `--group-by <mode>` | Group findings by detector, file, or package | detector |

## Default exclusions

The runner automatically excludes common non-source directories:

`node_modules`, `dist`, `build`, `.next`, `coverage`, `__pycache__`,
`.venv`, `vendor`, `target`, `bin`, `obj`, `out`, `.git`, `.desloppify`,
`*.min.js`, `*.min.css`, `generated`, `autogenerated`, `pb-*`

Add more with `--exclusions`.
